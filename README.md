#  C# Windows Forms 계산기 프로젝트 (과제 3)

본 프로젝트는 **MS 윈도우 계산기의 표준 모드**를 벤치마킹하여, 사용자 편의성(UX)과 데이터 동기화 로직을 강화한 C# 기반 계산기 애플리케이션입니다.

---

## 과제 3단계 주요 구현 기능

### 1. 실시간 수식 동기화 (Real-time Display Sync)
* **메인 입력창(`txtInputWindow`)**과 **보조 수식창(`txtOutputWindow`)**의 완벽한 데이터 연동.
* 숫자 입력, 연산자 선택뿐만 아니라 **편집(`del`, `CE`) 시에도 보조창 수식이 실시간으로 수정**됩니다.
* 예: `5 x 2 = 10` 상태에서 `del` 클릭 시 메인창은 `1`, 보조창은 `5 x 2 = 1`로 즉시 업데이트됩니다.

### 2. 정밀 편집 로직 (Del & CE Logic)
* **`del` (Backspace)**: `Substring` 메서드를 활용하여 마지막 한 글자만 제거합니다. 모든 글자가 지워질 경우 기본값 `0`으로 자동 복구됩니다.
* **`CE` (Clear Entry)**: 현재 입력 중인 숫자(Entry)만 초기화하며, 이전 연산 결과와 연산자는 메모리에 유지하여 작업의 연속성을 보장합니다.
* **상태 제어**: 편집 기능 수행 시 `isOperatorClicked` 플래그를 정교하게 제어하여, 지우기 작업 후 숫자를 입력할 때 화면이 의도치 않게 초기화되는 버그를 완벽히 해결했습니다.

### 3. 예외 처리 및 연산 고도화
* **자료형 확장**: `int` 대신 `double` 타입을 사용하여 소수점 연산(`5 ÷ 2 = 2.5`)의 정확도를 확보했습니다.
* **0 나누기 방지**: 분모가 0일 경우 `MessageBox`를 통한 안내 및 연산 중단 로직을 포함하여 런타임 에러를 방지했습니다.
* **중복 방지**: 소수점(`.`) 중복 입력 방지 및 부호(`+/-`) 전환 기능을 구현했습니다.

## 🛠 기술적 구현 디테일

### 🔹 공통 업데이트 엔진: `UpdateOutputWindow()`
코드의 중복을 줄이고 유지보수성을 높이기 위해, 모든 UI 변화를 감지하여 보조창을 다시 그려주는 전용 메서드를 구현했습니다.
* 연산자 입력 전/후 상태 구분 로직
* 결과 도출 후(`=`) 수식 수정 로직 포함

### 🔹 문자열 조작 로직
* `IndexOf('=')`와 `Substring`을 조합하여 이미 완성된 수식 문자열의 무결성을 해치지 않고 결과값 부분만 동적으로 수정합니다.
